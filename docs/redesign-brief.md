# 项目环境管家 — 重新设计简报（功能 + 交互全量清单）

> 用途：把这份文档整体交给一个 AI，让它**重新设计本产品的界面与交互体验**。
> 下面列的是「现状的功能与交互事实」+「要补齐的缺失功能」+「这是个 Chrome 扩展的形态约束」，**不是** UI 规范。
> 视觉、信息架构、布局、组件、流程编排都可以推翻重来，鼓励大胆改。

---

## 0. 给设计 AI 的话（先读这段）

- **这是什么**：一个 **Chrome MV3 浏览器扩展**（技术栈 WXT + React 19 + TypeScript + Tailwind v4），用来管理「公司多项目 / 多环境 / 多平台 / 多账号」的登录凭据，本地端到端加密、零知识，支持一键安全填充、生物识别解锁、多端同步。它**不是**普通网页 App，也不是桌面软件——它有几种浏览器扩展独有的「界面入口」和硬约束，必须照顾（见 **第 5 节**）。
- **你的任务**：基于 **第 3 节（全量功能）** 和 **第 4 节（主要交互）**，重新设计**界面与交互**。功能要全部覆盖，并补上 **第 6 节** 标注的缺失功能（尤其是「多端同步选择目录」）。
- **可以动的**：视觉风格、配色、信息架构、导航结构、布局、组件形态、交互流程的编排、文案。
- **不可削弱的**：**第 7 节** 列出的安全与功能硬约束（零知识加密、精确填充、最小权限、不联网等）。这些是产品立身之本，可以换更好的表达方式，但不能丢能力、不能降低安全性。
- **自由度**：除第 7 节外，**不要被现有 UI 束缚，自行设计，不需要逐屏还原现状**。现状描述只是为了让你知道"有哪些能力和交互需要被新设计承载"。

## 1. 产品定位与核心概念

**一句话**：给开发者/团队用的「项目环境凭据管家」——按个人/公司等工作区隔离首页和项目，把每个工程的入口链接、环境标签（dev/test/staging/prod）、Git 仓库、账号密码、两步验证码集中加密保管，并能在对应站点一键安全填充登录。

**核心数据层级（贯穿全产品，重设计需保留这套心智模型）**：

```
工作区 Workspace（个人 / 公司等）
 ├─ 首页看板 dashboard（多看板 + 磁贴排版/大小/配置/外观）
 └─ 项目/工程 Project
     ├─ 标签 tags[]、收藏 favorite、备注 note、颜色
     ├─ 说明文档 docs[]（多篇 Markdown）
     ├─ 待办 memos[]（文本 + 截止 + 紧急 + 完成）
     └─ 平台/链接 PlatformLink（主网址 url + 更多网址 urls[] + Git 仓库 + 环境标签 envKind/envName）
         └─ 账号 Account（备注名 label / 用户名 / 密码 / 备注 / TOTP 两步验证）
```
> 旧数据里的 Environment 会保留为兼容容器，并在迁移时把 kind/name 写入链接的环境标签。一个链接可挂多个账号；一个账号可带一个 TOTP 密钥。删除任何层级都会写「墓碑 tombstone」用于多端同步防复活。

**安全模型（决定了若干交互必然存在）**：主密码 → Argon2id 派生 KEK → KEK 包裹随机保险箱密钥 DEK → DEK 用 AES-GCM 加密整库。**落盘只有密文**，服务器/磁盘/其它程序都解不开。因此产品里必然有：解锁门禁、修改主密码、生物识别解锁、备份/同步引导（卸载会清数据，无备份即永久丢失）。

## 2. 用户能进入的「界面入口」（surfaces，扩展特有，先建立全局观）

| 入口 | 触发 | 承载什么 |
|---|---|---|
| **工具栏 Popup** | 点扩展图标 / `Alt+Shift+P` | 当前站点匹配账号、一键填充/打开登录、搜索、复制、收藏与最近使用、捕获横幅、同步/锁定 |
| **Options 全屏页** | 「管理全部」/独立标签页 | 主力界面：首页看板、项目详情、全局搜索、设置、导入导出、审计、文档、大屏、各类弹窗 |
| **锁屏 LockScreen** | 未初始化 / 已锁定 | 创建保险箱、主密码解锁、生物识别解锁、从同步服务器恢复 |
| **地址栏 Omnibox** | 地址栏输入 `env` + 空格 | 实时联想账号，回车「打开并登录」 |
| **右键菜单** | 网页/链接上右键 | 「保存到项目环境管家」→ 捕获录入 |
| **键盘快捷键** | `Alt+Shift+F/L/P` | 填充当前页 / 锁定金库 / 打开扩展（可在 `chrome://extensions/shortcuts` 改键）|
| **浮动备忘小组件** | options 页右下角 | 可拖动/折叠的待办悬浮窗，跨项目聚合 |
| **后台 Service Worker** | 无 UI | 所有加解密、消息路由、自动锁定、自动同步、剪贴板清空、登录捕获处理 |
| **卸载告知页** | 卸载扩展后自动打开 | 解释「卸载会清空本地数据」及如何用同步/备份恢复 |

## 3. 全量功能清单（穷举，按能力域分组）

### A. 保险箱与账户安全
- 创建保险箱（首次设主密码，≥8 位；生成永久 vaultId + 随机 DEK）。
- 主密码解锁 / 手动锁定 / 快捷键锁定（`Alt+Shift+L`）。
- **空闲自动锁定**（可配置 N 分钟，0=不锁不推荐；基于 chrome.idle + 定时器双保险）。
- 修改主密码（信封加密，仅重包 DEK，O(1)，数据密文不动）。
- **生物识别解锁**：Touch ID / Windows Hello（WebAuthn PRF），可注册多个设备、各自命名、可删除；**主密码始终保留作兜底**；平台不支持时禁用并提示。
- 清空整个保险箱（危险操作，二次确认，提示先备份）。
- 从同步服务器恢复保险箱（新设备 adopt，拉远端密文覆盖本机后用主密码解锁）。
- 会话保持：DEK 仅存内存 + `storage.session`，浏览器关闭即清；service worker 重启可恢复解锁态。

### B. 数据管理（工作区 / 项目 / 链接 / 账号）
- 工作区：新建、切换、重命名；每个工作区有独立首页看板、项目、文档和待办。旧数据自动迁移到“默认工作区”。
- 项目：新建/编辑/删除、标签（逗号分隔）、备注、收藏星标、**拖拽排序**、显示环境标签数/账号数。
- 环境标签：开发/测试/预发/生产/其它，作为链接属性展示；旧版环境容器继续用于兼容、移动和批量整理。
- 链接/平台：新建/编辑/删除，主网址 + **更多网址（多个，每行一个）**、环境标签、可挂 Git 仓库；额外网址用 +N 徽标展示。
- 账号：新建/编辑/删除，备注名/用户名/密码/备注/TOTP；密码可显隐、**一键生成 20 位强密码**。
- **Git 仓库小标签**：一键复制仓库地址 / 复制 `git clone` 命令（带分支）。
- 账号行操作：显隐密码、复制用户名、复制密码、打开并登录、编辑、删除。
- 通用对话框系统（样式化的 confirm/prompt，删除统一二次确认，danger 红色按钮）。

### C. 自动填充与登录（产品最核心的"省事"能力）
- **一键填充到当前页**：popup 中点账号「填充到当前页」，写入当前页登录表单。**严格精确 origin 匹配**（scheme+host+port 完全一致，绝不放宽子域），注入前二次复核实时 URL 防重定向误填。
- **智能定位字段**：自动找可见密码框 + 其前最近的用户名框，用原型 setter 赋值并派发 input/change 事件以兼容前端框架。
- **默认不自动提交**；可在设置显式开启「填充后自动登录」（仅 origin 完全一致才提交）。
- **账号选择器在扩展自己的 UI 里**（防页面点击劫持）；多账号匹配时列出由用户挑。
- **打开并登录（openAndFill）**：对带 URL 的条目新开标签页打开、等加载、校验最终 origin 后注入填充。未授权该站点时引导先授权。
- 快捷键「填充当前页」（`Alt+Shift+F`）：唯一匹配直填，否则打开 popup 选/解锁。
- **环境快速切换**：在某环境页面，一键跳到同项目其它环境的「同名链接 + 同路径」地址（如 dev→prod 同 `/orders/9`）。
- **登录捕获**：在已授权站点监听表单提交，抓到用户名+密码后提示「新增/更新」（角标 + popup 横幅），用户确认才存。
- 把当前网址加入已有链接（让该地址今后可被识别填充）。
- 收藏 / **最近使用** 快捷启动（当前站点无匹配时，popup 展示收藏账号与最近使用过的账号）。

### D. TOTP 两步验证
- 实时生成 6（–10）位验证码 + 周期倒计时（≤5 秒变红），点击复制。
- 手动填 base32 密钥或 `otpauth://` URI；**二维码图片本地识别**自动回填（jsQR，无联网）。
- **Google Authenticator 迁移码批量导入**（解析 `otpauth-migration://` protobuf，批量建账号）。
- CSV 导入时识别并带入 TOTP（Bitwarden/1Password 等）。
- **验证码墙磁贴**：聚合所有带 TOTP 的账号，默认遮蔽，眼睛切换显示，环形倒计时。
- 验证码在 popup 条目、options 账号行、搜索结果行多处内嵌展示。

### E. 全局搜索
- 账号级摊平搜索，空格分词 AND 子串匹配（项目/环境/链接/网址/账号名/用户名/备注）。
- 入口多处：options 侧栏搜索、popup 搜索、**地址栏 omnibox**、首页搜索磁贴。
- 结果行可显隐密码、复制用户名/密码、复制 TOTP、打开并登录。

### F. 首页看板 / 仪表盘（homarr 风格，自研零依赖网格）
- 自由定位网格引擎：磁贴 (x,y) 锚点 + w×h 占格，**拖拽移动、拖拽缩放、碰撞让位、纵向重力压缩**、拖拽落点虚位预览、拖到边缘自动滚动。
- **响应式断点**（按容器宽度 4/3/2 列自适应，窄屏由基准布局派生）。
- **多看板**（页签切换，新建/重命名/删除，每个看板独立磁贴集合 + 独立外观）。
- **编辑模式 vs 浏览模式**切换（编辑态才显示手柄/添加/外观/看板增删）。
- **16 种磁贴**：统计 stats、待办 todos、日历 calendar、快捷入口 launcher（app 图标墙）、全局搜索 search、验证码墙 totp、密码健康度 health、最近使用 recent、Git 仓库 repos、标签云 tags、文档速览 doc、近期改动 changed、备份/同步状态 backup、时钟 clock、天气 weather、图片/图表 image。
- 每卡配置面板（自定义标题、限定项目、选文档、仅收藏、默认揭示验证码等）。
- **外观面板**：背景（无/预设渐变 5 选/上传图）、磁贴不透明度、磁贴模糊（玻璃拟态），随看板独立存储。
- **项目大屏模式 BigScreen**：全屏只读展示单项目（统计/链接账号/clone 命令/待办/文档），Esc 退出。
- 主题切换（明/暗/跟随系统）。

### G. 项目说明文档（Markdown）
- 每项目多篇 Markdown 文档：新建、删除、**导入 .md/.txt（可多选）**、改标题、改正文。
- 阅读/分屏/源码三态，**自动保存**（防抖 + 串行写入，顶部显示保存状态）。
- 渲染：marked + DOMPurify 消毒；**mermaid 流程图懒渲染**；标题锚点 + 目录（TOC）；**文档内搜索高亮跳转**（Ctrl/Cmd+F）。
- 右侧栏「说明」标签页内联多文档浏览。

### H. 待办 / 备忘
- 项目待办：新增（文本 + 截止 + 紧急）、勾选完成、切换紧急、删除。
- 截止「逾期/今天/明天/N 天后」标签；逾期或紧急未完成会标红抖动提醒。
- 排序（未完成优先 → 截止升序 → 创建时间）。
- **浮动备忘小组件**（右下角，可拖动/折叠，跨项目聚合，徽标计数，有提醒整体变红）。
- 待办磁贴、日历磁贴（按截止日聚合，翻月，点某天看当天待办）。
- 右侧栏「待办」标签页。

### I. 安全审计
- 本地检测：弱密码（长度/字符类别）、重复使用、超半年未更新；输出逐条问题 + 计数。
- 审计弹窗（统计卡 + 问题列表）、密码健康度磁贴（评分环）。全程本地不外传。

### J. 多端同步（端到端加密，后端只见整库密文）
- **7 种后端**：自托管服务器、WebDAV、GitHub、GitLab、Google Drive、OneDrive、群晖 Synology NAS。一个保险箱可同时配多个目标。
- 每目标：**双向合并同步**（拉→解密→按 id+时间戳+墓碑三路合并→重加密→乐观并发推回，冲突自动重试）、**强制推送**（本地覆盖远端）、**强制拉取**（远端覆盖本地）。
- 「同步全部已启用目标」；**内容修改后自动同步**（可关，2.5s 防抖）。
- **异库合并**：远端是不同主密码的保险箱时，要求输入远端主密码再合并。
- 网盘 **OAuth 授权**（Authorization Code + PKCE，client_id 用户自带，只存 refresh_token 加密入库）。
- 群晖 **两步验证 OTP + 受信设备令牌绑定**（首次 OTP 绑定，之后免 OTP）。
- 保存前 **连通性/凭据预检**；**Git 公开仓库安全警示**（离线爆破风险，需二次确认）。
- 同步目标增删改（脱敏视图、密钥留空沿用旧值）、每目标运行状态展示（上次同步/错误/未授权标记）。
- 运行时按需申请目标所需 host 权限。

### K. 导入导出与备份
- 导出：**加密整库备份**（全部工作区、首页看板、项目文档、待办；独立备份密码）、明文 JSON、明文 CSV（明文导出前危险确认）；可按当前工作区项目选择导出范围。
- 导入：加密备份、明文 JSON、本扩展 CSV；**迁移**：Chrome 密码 CSV、Bitwarden CSV、1Password CSV、Google Authenticator 迁移码/二维码。
- 合并模式（按名称去重增量追加）/ 替换模式（清空后导入，黄色警告）。
- 文件或粘贴两种来源。
- **首次创建后的备份引导**（强提示二选一：导出加密备份 / 开启同步）。
- **久未备份提醒条**（>14 天且未开同步时首页顶部提醒，可稍后静默 7 天）。

### L. 设置项（穷举）
主题（明/暗/系统）、填充后自动提交开关、待办悬浮窗显隐、首页天气联网开关、自动锁定时长、修改主密码、生物识别注册/移除、多端同步目标管理、内容修改后自动同步开关、清空保险箱。

### M. 剪贴板安全
- 复制用户名/密码/TOTP 后提示「25 秒后自动清空」，由 **offscreen 文档**可靠计时清空（不受 service worker 回收影响）。

## 4. 主要交互方式（精选，体现产品特色的关键流程）

> 这些是最该被新设计照顾好的核心交互。重点是**触发方式 → 反馈**，具体长什么样由你设计。

1. **解锁门禁**：打开任何 surface，未解锁先进锁屏（创建/主密码/生物识别/从同步恢复四选一）。生物识别一次系统弹窗即解锁。
2. **当前站点一键填充**：在登录页点扩展图标 → popup 顶部直接列出该站点匹配账号 → 点「填充到当前页」→ 字段被填好（默认不提交）→ popup 关闭。origin 不符时不显示填充按钮/阻止填充。
3. **打开并登录**：从任意位置（popup/搜索/omnibox/launcher 磁贴）选一个账号 → 新标签打开其网址并自动填充。首次需对该站点授权（运行时权限弹窗）。
4. **登录捕获**：用户在已授权站点正常登录 → 角标出现红点、popup 顶部横幅「检测到新账号，保存？」→ 一键保存/更新/忽略。
5. **环境切换**：看着 dev 的某页面，一键跳到 prod 的同路径页面。
6. **看板编辑**：进编辑模式 → 拖磁贴移动/缩放、实时让位与落点预览 → 添加 16 种卡片之一 → 配置每卡数据绑定 → 完成回到浏览态。多看板页签切换。
7. **同步目标配置**：添加目标 → 选类型 → 填地址/凭据（网盘走 OAuth 授权、群晖走 OTP 绑定）→ 预检（公开仓库会警示）→ 保存。之后每目标可同步/强推/强拉。
8. **TOTP 录入**：账号编辑里手填密钥，或选二维码图片自动识别；批量从 Google Authenticator 迁移码导入。
9. **文档编写**：项目说明里写 Markdown（含 mermaid），自动保存，阅读态有目录与文内搜索。
10. **备份引导**：首次创建后强提示去备份/开同步；久未备份首页顶部提醒。
11. **全局搜索**：侧栏/popup/地址栏 `env`/搜索磁贴四处可搜，结果可直接复制或打开登录。
12. **复制即焚**：任何密码/TOTP 复制后 25 秒自动清空剪贴板。

## 5. 这是 Chrome 扩展 — 必须照顾的特殊交互方式与约束（重点提醒设计 AI）

> ⚠️ 这一节是和"普通网页/桌面 App"最不同的地方。新设计的导航、流程、组件都要在这些约束内成立。

**多入口形态（一个产品，多块画布）**
- **Popup 是临时的、会被销毁**：点扩展图标弹出的小窗，**一旦失焦就关闭**（点别处、切标签、系统弹窗弹出都会关）。所以 popup 适合"当前站点的快捷操作"，**不能承载需要长时间停留或会触发系统弹窗的流程**；尺寸也受限（窄而短）。
- **Options 是常驻全屏页**：主力管理界面放这里。重设计时要明确「哪些能力放 popup（轻、快、当前页相关）、哪些放 options（全、重、管理类）」。
- **WebAuthn（生物识别）只能在 options/独立标签页里跑**，绝不能在 popup（指纹系统弹窗会让 popup 关闭中断）。现状里 popup 点生物识别会自动转去 options 页完成——新设计要保留这种「转交」思路。
- **首次授权站点的权限弹窗会关闭 popup**：所以「打开并登录」首次授权设计成在 options 页完成的中转流程（密码不经 URL 传递，只传账号 id）。

**权限与安全边界（影响"能做什么"）**
- **最小权限，不申请 `<all_urls>`**：基础只有 storage / activeTab / scripting / idle / contextMenus / offscreen / clipboardWrite / identity。对具体站点的访问权是**用户运行时按需授权**（`chrome.permissions.request`，会弹系统确认框）。所以任何"打开并填充某站点""自托管/WebDAV 同步到某域名"都伴随一次授权交互——新设计要把这个授权步骤表达清楚、顺滑。
- **内网开发环境常是明文 http**：授权要支持 http（不能只限 localhost）。
- **service worker 约 30 秒空闲被回收**：所以 DEK 存 `storage.session`、剪贴板定时清空交给 **offscreen 文档**而非后台计时器。设计上意味着"后台不是一直活着的"。
- **严格 CSP（MV3）**：禁内联脚本、**禁远程代码**。图标用首字母而非远程 favicon、背景图/上传图以 dataURL 内联存进加密库、mermaid 与 Argon2(wasm) 都在本地跑。**默认不联网**（唯一例外是天气磁贴，且需用户显式开启才向第三方发请求）。新设计**不要引入任何需要联网/远程资源/CDN 的视觉方案**。

**扩展专属的交互入口（都要在新设计里有归属）**
- **Omnibox**：地址栏输入 `env` + 空格进入搜索，回车打开并登录。
- **右键菜单**：网页/链接上「保存到项目环境管家」。
- **键盘快捷键**：填充当前页 / 锁定 / 打开扩展（可改键）。
- **角标 Badge**：用扩展图标角标提示待处理（如捕获到新登录显示红色 `1`）。
- **idle 自动锁定**：空闲自动锁，所有 surface 解锁态会因此被打断回锁屏。
- **卸载即清数据**：Chrome 卸载扩展会清掉本地存储，**没有服务器副本**（零知识）。因此「备份/同步引导」「卸载告知页」是刚需，不是可选装饰——这点深刻影响首启与设置的信息架构。
- **多入口各自初始化主题**：popup / options 是不同的页面上下文，主题、状态各自加载（解锁前跟随系统，解锁后按设置）。

## 6. 当前缺失 / 不完整、重设计要补齐的功能

> 用户点名的「多端同步选择目录」是第一优先。其余按价值排列，请一并纳入新设计。

### ⭐ 6.1 多端同步：远端目录 / 路径选择（务必补齐）
现状是**几乎没有目录选择能力**，落点全靠手填字符串、规则还各不相同：

| Provider | 现状 | 期望（重设计补齐） |
|---|---|---|
| **Google Drive** | 只能填「文件名」，密文被**写死进 `appDataFolder` 隐藏应用目录**，用户在 Drive 网页端看不到、也改不了位置 | 允许**选择/新建普通文件夹**作为存放位置（理想：弹出 Drive 文件夹选择器；至少可填文件夹路径），并在 Drive 网页端可见可管理 |
| **OneDrive** | 只能填文件名，密文写死进应用专属 `approot` | 允许指定 OneDrive 内任意路径/文件夹 |
| **自托管服务器** | 协议写死 `/v1/vault`，一个 serverUrl+token 只能存单一保险箱 | 支持**子路径 / 多库**（同一服务器放多个库、自定义路径） |
| **GitHub / GitLab** | 可填 `filePath`（含子目录）但**全靠手填、无校验、无分支/路径浏览**；多级 group 命名空间易混淆 | 加**分支选择、路径浏览/校验**；理顺 owner/namespace 与 projectId 的关系 |
| **WebDAV** | 目录塞在 url、文件名塞在另一个字段，规则与群晖不一致 | 统一的「目录 + 文件名」表达，最好能**浏览远端目录** |
| **群晖 Synology** | 目录+文件名合并写进单个字段、且必须以共享文件夹名开头 | 统一表达 + **远端目录浏览/选择**（群晖 FileStation 有列目录 API，可接） |

**通用诉求**：提供一个**远端目录「浏览 / 列举 / 选择」器**，让用户不必手拼路径字符串；OAuth 目标补「测试连通 / 校验落点可写」反馈（现状网盘 preflight 只验 token、不验路径）。

### 6.2 其它值得补齐的缺口（按价值）
- **窄屏无法编辑看板布局**（<1024px 只读，且派生布局不落库）——重设计应支持窄屏/移动可编辑或更合理的响应式编辑模型。
- **审计结果不能直达修复**：审计/健康度磁贴的问题账号只能看，点不进去改密码——应支持一键跳到对应账号编辑。
- **改主密码不能顺带升级 KDF 参数**（底层支持、UI 没暴露）；创建时也不能选 KDF 强度。
- **备份引导/提醒的「开启同步」只打开设置页，不定位到同步分区**——应直达并高亮同步配置。
- **加密备份与明文备份同为 `.json` 后缀**，靠内部字段区分，用户凭文件名分不清——应在文件名/导出体验上区分清楚。
- **导入合并按名称去重、密码不同会被静默丢弃**，且 Bitwarden 非 login 条目被静默忽略——应给**冲突提示/忽略计数**。
- **天气磁贴城市无持久配置**（只能卡内即时输入，换设备/刷新行为不明）——应可配置默认城市/单位。
- **最近使用记录仅本机、不跨端**——可考虑纳入同步或至少说明。
- **登录捕获**对分步登录、iframe、shadow DOM 表单可能漏抓/抓错——可在新交互里给"手动校正抓取结果"的机会。
- **TOTP 只支持 TOTP、忽略 HOTP**；迁移码的非标准算法/位数会被静默降级——可提示不支持项。
- 浮动备忘没有"就地隐藏"入口（只能去设置关）；文档列表无搜索/排序。
- 项目「收藏星标」目前只是标记、不影响排序——与一般"收藏置顶"直觉不符，可重新定义。

## 7. 重设计的硬约束（不可削弱，可换更好的表达，但不能丢能力 / 不能降安全）

1. **零知识 / 端到端加密**：主密码派生密钥、本地加密、**落盘与同步都只有密文**，服务器无法解密。任何新设计不得把明文外传或弱化加密。
2. **填充安全**：精确 origin 匹配（不放宽子域）、**默认不自动提交**、**账号选择必须在扩展自己的 UI 内**（防点击劫持）、注入前复核实时 URL。
3. **最小权限**：不申请 `<all_urls>`/常驻 host 权限，站点访问运行时按需授权。
4. **不联网 / 无远程代码**：除用户显式开启的天气外不发任何网络请求；不引入 CDN/远程字体/远程脚本/远程图片（CSP 禁止，也是审核与隐私要求）。
5. **生物识别**：必须在 options/标签页执行；**主密码始终保留作兜底**，丢失授权器不能锁死保险箱。
6. **数据安全后路**：保留备份/同步引导与卸载告知（卸载即清、无服务器副本）。
7. **多设备一致性**：删除走墓碑、合并按 id+时间戳——重设计不应破坏多端合并的正确性。
